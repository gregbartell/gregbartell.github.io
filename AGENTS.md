# AGENTS.md

## Site Shape

This is a static Washington license plate collection site. Preview normal site
work by opening `index.html` directly in a browser; do not require a local HTTP
server for routine development.

Browser code must stay browser-native. Node is for tests and maintainer scripts
only, not for website runtime code.

## Dependency Boundary

This repo is intentionally dependency-light. Do not add or rely on new runtime,
build, test, or maintainer dependencies without an explicit project decision.

Do not introduce a package manager, bundler, framework, build step, or dev server
for normal site work. Keep the site usable by opening `index.html` directly.

## Catalog Source Of Truth

The Plate Catalog lives in `scripts/plate-catalog.js`. Put catalog facts there:
categories, plate entries, photo statuses, selected assets, and image kinds.

`index.html`, `styles.css`, and `scripts/plate-page.js` are presentation code.
Do not hard-code catalog facts into the page markup or presentation layer.

Use the catalog language in `CONTEXT.md` when naming concepts or writing docs.

## Images And Thumbnails

Full-size catalog JPEGs live under `pics/<category>/`. Selected asset paths in
`scripts/plate-catalog.js` are written relative to `pics/`, for example
`parks/state_parks.jpg`.

`pics/thumbs/` contains committed site assets generated by
`scripts/make_thumbs.py`, which uses Pillow. Thumbnail paths mirror full-size
paths under `pics/thumbs/<category>/`.

`reference/` holds source/reference imagery for design and cleanup work. It is
not the catalog image root; selected catalog assets live under `pics/` and are
referenced from `scripts/plate-catalog.js`.

Selected assets must have both a full-size JPEG and a matching thumbnail. A
plate with `photoStatus: "missing"` must use `asset: null`.

Unselected local images under `pics/` may be valid collection material or older
alternate photos. `scripts/audit_catalog.js` reports them as informational
notices, not catalog failures; do not delete them just because they are
unselected.

Use lowercase snake_case for category IDs and image folders. Use lowercase
kebab-case for plate IDs. Keep catalog order alphabetical by category and plate
title, with Miscellaneous last.

## Common Tasks

Add or replace a selected plate photo:

1. Add the full-size JPEG under `pics/<category>/`.
2. Run `python3 scripts/make_thumbs.py`.
3. Update the selected asset in `scripts/plate-catalog.js`.
4. Run `node scripts/audit_catalog.js`.

Add or update a catalog entry:

1. Update `scripts/plate-catalog.js`.
2. Run `node scripts/audit_catalog.js`.

Change presentation:

1. Edit `index.html`, `styles.css`, or `scripts/plate-page.js`.
2. Open `index.html` directly in a browser and inspect the result.

## Verification

After catalog fact changes, run:

```sh
node scripts/audit_catalog.js
```

After catalog logic changes, run:

```sh
node scripts/plate-catalog.test.js
node scripts/audit_catalog.test.js
```

After adding full-size JPEGs under `pics/`, run:

```sh
python3 scripts/make_thumbs.py
node scripts/audit_catalog.js
```

After presentation or CSS changes, open `index.html` directly in a browser and
inspect manually.

Do not invent `npm` commands unless a `package.json` is added through an
explicit project decision.

## Presentation And Design

For visual design and visible copy tone, follow `.stitch/DESIGN.md`. Treat that
file as a reference unless intentionally changing the design system or
visible-copy direction.

When changing presentation behavior, preserve existing keyboard access, modal
focus/Escape behavior, lazy thumbnail loading, and catalog-derived alt text.

## Git

Use conventional commit titles: `<type>(<scope>): <subject>`. Keep the subject
imperative and 50 characters or less. Wrap commit body lines at 72 characters or
less.

Keep commits atomic: one logical repo change per commit.

## Agent Workflow Docs

Use these docs when working on planning, triage, issues, or agent workflow:

Issues and PRDs are tracked as local markdown files under `.scratch/`; external
PRs are not a triage surface. See `docs/agents/issue-tracker.md`.

The default five-role triage vocabulary is used unchanged. See
`docs/agents/triage-labels.md`.

## Domain Docs

This is a single-context repo with root `CONTEXT.md` and root `docs/adr/`. See
`docs/agents/domain.md`.

Update `CONTEXT.md` or `docs/adr/` only when a change intentionally alters
domain vocabulary, catalog structure, or architectural decisions.
